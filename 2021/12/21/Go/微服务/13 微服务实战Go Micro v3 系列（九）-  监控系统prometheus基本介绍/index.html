微服务监控系统promethues基本介绍

  • 是一套开源的监控&报警&时间序列数据库的组合
  • 基本原理是通过HTTP协议周期性抓取被监控组件的状态
  • 适合Docker、Kubernetes环境的监控系统

微服务监控系统promethues架构

微服务监控系统promethues重要组件

  • Prometheus Server：用于收集和存储时间序列数据
  • Client Library：客户端库组成相应的metrics并暴露给Prometheus server
  • Push Gateway：主要用于短期的jobs
  • Exporters：用于暴露已有的第三方服务的metrics给Prometheus
  • Alertmanager：从Prometheus server端接收到alerts后，会进行去除重复，分组，并路由到对收的接受方式，发出警报

微服务监控系统promethues工作流程

  • Prometheus server定期从配置好的jobs/exporters/Pushgateway中拉数据
  • Prometheus server记录数据并且根据警报规则推送alert数据
  • Alertmanager根据配置文件，对接收到的报警进行处理，发出告警
  • 在图形界面中，可视化采集数据

微服务监控系统promethues相关概念-数据模型

  • Prometheus中存储的数据为时间序列
  • 格式上有metric的名字和一系列的标签(键值对)唯一标识组成
  • 不同的标签则代表不同的时间序列

微服务监控系统promethues相关概念-metric(指标)类型

  • Counter类型：一种累加的指标，如：请求的个数，出现的错误数等
  • Gauge类型：可以任意加减，如：温度，运行的goroutines的格式
  • Histogram类型：可以对观察结果采样，分组以及统计，如：柱状体
  • Summarg类型：提供观测值的count和sum功能，如：请求持续时间

微服务监控系统promethues相关概念-instance和jobs

  • instance：一个单独监控的目标，一般对应于一个进程
  • jobs：一组同种类型的instances(主要用于保护可扩展性和可靠性)

微服务监控系统grafana看板

  • 拥有丰富dashboard和图标编辑的指标分析平台
  • 拥有自己的权限管理和用户管理系统
  • Grafana更适合用于数据可视化展示
